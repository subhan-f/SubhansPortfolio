# Mastering Framer Motion Animations in React

Animations can make or break a user experience. When done right, they guide users, provide feedback, and add a layer of polish that separates great apps from good ones. **Framer Motion** makes this easier than ever.

## Why Framer Motion?

Compared to CSS animations or other libraries, Framer Motion offers:

- **Declarative API** — Describe what you want, not how to achieve it
- **Layout animations** — Animate between layouts automatically
- **Gesture support** — Tap, drag, hover, and more
- **Exit animations** — Animate components as they unmount

## Basic Animations

The simplest animation uses the `motion` component:

```tsx
import { motion } from 'framer-motion';

function FadeIn({ children }) {
  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: 0.6 }}
    >
      {children}
    </motion.div>
  );
}
```

## Scroll-Triggered Animations

One of the most impactful patterns is revealing content as users scroll:

```tsx
import { motion, useInView } from 'framer-motion';
import { useRef } from 'react';

function ScrollReveal({ children }) {
  const ref = useRef(null);
  const isInView = useInView(ref, { once: true });

  return (
    <motion.div
      ref={ref}
      initial={{ opacity: 0, y: 50 }}
      animate={isInView ? { opacity: 1, y: 0 } : {}}
      transition={{ duration: 0.8, ease: 'easeOut' }}
    >
      {children}
    </motion.div>
  );
}
```

## Staggered Children

For lists and grids, staggering children creates a beautiful cascade effect:

```tsx
const containerVariants = {
  hidden: {},
  visible: {
    transition: {
      staggerChildren: 0.1,
    },
  },
};

const itemVariants = {
  hidden: { opacity: 0, y: 20 },
  visible: { opacity: 1, y: 0 },
};
```

## Performance Tips

1. **Use `layout` prop sparingly** — It triggers layout recalculations
2. **Prefer `transform` properties** — `x`, `y`, `scale`, `rotate` are GPU-accelerated
3. **Use `will-change`** on heavy animations
4. **Avoid animating `width`/`height`** — Use `scale` instead

## Conclusion
